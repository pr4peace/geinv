How should we build this?

Three approaches for the Good Earth investment tracking platform — pick what feels right

A

Lightweight Web App ★ Recommended

Stack: Next.js + Supabase (database & file storage) + Claude AI + Resend (email)

Upload a scanned agreement → Claude extracts all fields → data saved → reminder schedule auto-generated → emails sent on schedule. Single deployable app, minimal infrastructure.

Pros

  • Fast to build and deploy
  • Low running cost
  • Easy to extend later (WhatsApp, multi-user)
  • Supabase handles auth, storage, DB in one

Cons

  • Depends on Supabase (managed service)
B

Custom Full-Stack App

Stack: React + Node.js/Express + PostgreSQL + cron jobs + SendGrid

Fully custom backend and database. More control over every layer — useful if you want self-hosted or have specific IT constraints.

Pros

  • Full control, self-hostable
  • No dependency on managed services

Cons

  • Longer to build
  • More infrastructure to manage
  • Overkill for single-user tool
C

No-Code + AI Automation

Stack: Airtable (database) + Make.com (automation) + Claude API

Use existing no-code tools for data storage and automation. Good for a quick prototype but limited in customising the calendar and dashboard experience.

Pros

  • Fastest to get started
  • No coding needed for logic

Cons

  • Limited custom UI (calendar, dashboard)
  • Gets expensive as data grows
  • Hard to extend meaningfully
